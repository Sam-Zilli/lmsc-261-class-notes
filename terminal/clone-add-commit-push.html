Clone, Add, Commit, and Push from terminal

What are these commands?

Git is a version control system that allows you to manage and track changes in your code. The commands we will cover are used to interact with a Git repository, making it easy to manage your project.

1. Git Clone

The 'git clone' command is used to create a local copy of a remote repository. This is typically the first step when you want to start working on an existing project.

NOTE: Make sure you are in the directory where you want this repo to be cloned before you run the command!

1
git clone https://github.com/username/repository.git

This will download the repository from GitHub (or another Git hosting service) to your local machine, allowing you to make changes.

After you've cloned your repo and made changes to it, it's time to do the three-step process to make those changes reflect on GitHub.com

2. Git Add

The 'git add' command is used to stage changes you’ve made to your files so that they can be committed to the repository. You need to add files before committing them.

2
git add .

The period ('.') stages all modified files in the current directory and its subdirectories. You can also add specific files like this:

3
git add index.html

3. Git Commit

The 'git commit' command is used to save your staged changes to the local repository. You should always include a message explaining what changes you’ve made.

4
git commit -m "Added new feature"

The -m flag allows you to add a commit message inline. This helps you and others understand what changes were made in that commit.

4. Git Push

The 'git push' command uploads your local changes to the remote repository (like GitHub). This is how you share your changes with others on the project.

5
git push

Here, origin refers to the default name for the remote repository, and main is the branch you are pushing to. If you're working on a different branch, you can replace main with your branch's name.

Here’s a brief recap:

git clone - Create a copy of a repository

git add - Stage changes for commit

git commit - Save your changes locally (takes a snapshot)

git push - Upload changes to the remote repository

Once you push, your changes are reflected in the remote repository (Github.com)
