UNDERSTANDING FILE PATHS

What is a File Path?

A file path is the location of a file or directory on your computer. It shows the hierarchy of directories (folders) leading to a specific file.


In the terminal, file paths are used to navigate to different files and directories.



File paths can be absolute or relative:

  • Absolute Path: An absolute path starts from the root directory and specifies the complete path to a file or directory. It is the full path to the file or directory from the root.
  • Relative Path: A relative path specifies the location of a file or directory in relation to the current directory.

Examples of File Paths

1. Absolute Path

An absolute path gives the complete address of a file starting from the root directory. It does not depend on the current working directory.

Example: The absolute path to a file might look like this:

/home/user/Documents/file.txt

Here, /home/user/Documents/ is the path to the directory, and file.txt is the file located inside that directory. The path starts from the root directory /.

2. Relative Path

A relative path is based on the current directory. It does not start from the root but from the current directory where you are located.

Example: If you are currently in the /home/user/ directory and you want to access a file in the Documents folder, the relative path would be:

Documents/file.txt

The relative path Documents/file.txt starts from the current directory and does not include the root path.

3. Navigating with cd and File Paths

When you use the cd command, you can navigate using either absolute or relative paths:

Using an Absolute Path:

$ cd /home/user/Documents

This moves you directly to the Documents directory from anywhere on the system.

Using a Relative Path:

$ cd Documents

If you are already in the /home/user/ directory, the command cd Documents takes you to the Documents directory inside the current directory.

Special Characters in File Paths

Some special characters are used in file paths to represent specific locations:

  • ~ - Represents the home directory of the current user. For example, cd ~/Documents takes you to the Documents folder in your home directory.
  • . - Represents the current directory. For example, ./file.txt refers to a file named file.txt in the current directory.
  • .. - Represents the parent directory. For example, cd .. moves you up one level in the directory tree.

File Path Examples in Terminal

1. Moving to the Home Directory

Using the cd ~ command moves you to your home directory:

$ cd ~

/home/user

2. Moving Up One Directory

The cd .. command moves you up one directory level:

$ cd ..

/home

3. Using a Relative Path

If you are in the /home/user/ directory and want to access a file in the Documents directory, use the relative path:

$ cd Documents
